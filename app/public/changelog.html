dojo-sandbox changelog

A brief history of changes.

Jan 2013

  • Fixed 'Fork' so it retains your login namespace and doesn't dump new bucket in 'public' if you are logged in.
  • Built and deployed dojo 1.8.2, 1.8.3.
  • Hid the 'Layers' controls, they don't make sense since we moved to the 1.7.x build system.
  • Note: the 'Cannot save - Do you want to fork?' dialog that appears when you are looking at someone else's namespace (e.g. a template) and try to Save does not work. Please just close that dialog and hit Fork manually for now.

Jan 2013

  • Added Templates. 'template' is a special namespace containing example buckets you can use as a base for your development. Combined with the Fork ability, they make starting a more complex bucket much easier. Just select a template name from the 'Template' dropdown in the toolbar to go to a template, edit it and Run it as much as you want, and Fork it to create your own copy of it.
  • Fixed login? You ought to be able to sign up and log in again.

Nov 2012

  • Added a quick n dirty how-to video.
  • Login and user creation may be broken at present.

Sept 2012

  • Big overhaul to use dojo 1.8 under the covers.
  • Introduced dojo versions 1.7 and 1.8, with dgrid (and put-selector and xstyle dependencies).
  • Working on Save, Delete and Fork operations.

Sunday August 28 2011

  • Ability to register and log on as a user.
  • Added this changelog.

Initial version

  • dojo versions 1.4.3, 1.5.0, 1.5.1, 1.6.0, 1.6.1
  • Selection of layers from the 'standard' build profile.
  • Saving of public buckets, save as new version.
  • Launch for debug in new window.
